# HEARTBEAT.md

## Core Rule
If nothing needs intervention, reply exactly:
`HEARTBEAT_OK`

## JobOps Heartbeat Priorities
Check in this order:
1. review queue overload or drift
2. high-signal opportunities lacking shortlist visibility
3. ambiguous staging vs canonical tracker state
4. recurring parser/noise issues that need escalation to JT7 Platform

## Speak only when
- there is a real blocker to job-search momentum
- review noise is hiding good opportunities
- canonical tracker cleanliness is at risk
- a platform issue needs explicit escalation

## Stay quiet when
- the state is materially unchanged
- the issue is already known and recently surfaced
- there is no concrete next move
