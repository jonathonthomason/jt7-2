## Additional durable updates
- Jonathon explicitly replaced the previous 4-bot idea with a new required bot set: `platform`, `job ops`, `life`, and `ta knowledge bot`.
- The planning bot should no longer be treated as part of the target live 4-bot surface model unless Jonathon later reintroduces it.
- JT7 platform architecture must treat all bots/threads as conversation surfaces on one unified platform: one engine, one gateway, one shared brain, no forked systems, and no duplicated logic.
- Platform agent is the control plane and owns routing, system rules, orchestration, and decisions about where work should go.
- Domain agents are worker agents; each thread defaults to its assigned domain agent and should not behave like a general assistant.
- Required platform behavior: determine whether a request belongs to the current thread domain, execute in-thread when it fits, and create an explicit structured handoff when it does not.
- Handoffs must be explicit, explain why work is moving, preserve traceability, package context cleanly, and avoid silent domain switching.
- Context model is now explicitly split into shared global context (user goals, system rules, major decisions, identity, priorities) versus thread-local context (active tasks, recent actions, domain-specific state), and these must not be mixed blindly.
- The current JT7 platform restructuring directive is to refactor the system into a real multi-surface product platform with consistent thread structure, reusable logic, modular agents, clear boundaries, predictable routing, and shared memory across domains.
